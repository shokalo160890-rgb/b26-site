Session-State vs Long-Term Memory

Session-state нужен для текущей задачи. Long-term profile нужен только для устойчивых повторяющихся паттернов.

Session-state

Хранит временные требования: формат, язык, срочность, текущую цель и корректировки.

Long-term profile

Хранит подтвержденные предпочтения: прямота, структура, code-first режим, tolerance к уточнениям.

Conflict handling

При противоречии поле не перезаписывается сразу. Уверенность понижается, новое значение держится временно.
