Case · Personalization

Adaptive User Profile Architecture

Гибридная система обновляемого поведенческого профиля для LLM-ассистентов.

Задача

Создать профиль, который адаптируется по текущим инструкциям, session-state и устойчивым предпочтениям, без жёстких психологических ярлыков.

Моя роль

Полная архитектура, правила обновления, метрики качества и privacy-контроли.

Стек

explicit_rules, session_state, long_term_profile, confidence ≥ 0.70, evidence_count, contradiction_rate, privacy-by-design.

Ключевые решения

Текущая инструкция сильнее истории, drift monitoring, rubric-based evaluation, data minimisation, pseudonymisation.

Результат

Операционная модель персонализации: прямота, структура, code-first режим, контроль противоречий и объяснимое обновление профиля.

Learnings

Персонализация должна быть операционной, а не диагностической.
